# 🚀 Генеральная доктрина `MAKE_OP`: Полный Свод Знаний

### I. Первоисточник: Где брать Истину?
Сигнатуры — наше всё. **Никакой отсебятины.** Вот проверенный путь поиска:
1.  **YAML-конфиги Op-Plugin (`op_plugin_functions.yaml`)**:
    *   Ищи `func: npu_имя_операции(...) -> ...`.
    *   Здесь ты найдёшь **каноничные типы аргументов** (`Tensor`, `Scalar`, `bool`, `int`) и значения по умолчанию. Это твой главный ориентир.
2.  **JSON-схемы (`torch_npu_OpApi_schema_all.json`)**:
    *   Ищи `"func: npu_имя_операции..."`. Ключевое поле `exec: aclnnXxxYyy` даёт тебе **точное имя aclnn-функции**, которую нужно реализовать.
3.  **Заголовочники CANN (`*.h`)**:
    *   Итоговая инстанция: `ACLNN_API aclnnStatus aclnnCatGetWorkspaceSize(...)`. Копируем **1-в-1**, соблюдая порядок аргументов. Пробелы перед `*` и `&` удаляем.

### II. Анатомия aclnn: Двухтактный Двигатель
Помни: aclnn-операция — это две функции, а не одна:
1.  **`GetWorkspaceSize`**: Вызывается первой. `torch_npu` ожидает, что мы здесь **создадим Executor** и запросим временную память (мы всегда просим `0`).
2.  **Основная функция (`DEFINE_ACLNN_OP`)**: Вызывается второй. Получает созданный Executor и выполняет код из `MAKE_OP`.

**Ключевое Правило:** Сигнатуру `GetWorkspaceSize` **менять нельзя**. Если в ней нет выходного тензора (как `gradInput` в старых версиях `GluBackward`), то мы не можем использовать `MAKE_OP` и оставляем операцию как `UNIMPLEMENTED`.

### III. Синтаксис `MAKE_OP` и Магия Типов

**Базовый шаблон:**
```cpp
MAKE_OP(aclnnИмяОперации(
    [модификатор] ТипАргумента имя_аргумента,
    ...
    uint64_t* workspaceSize, aclOpExecutor** executor  // Всегда в конце!
) {
    // Твоё тело. Пиши чисто, без exec-> (кроме примения exec к тензорам).
})
```

#### Таблица конвертации `acl*` → C++ (детальная версия)

| Тип в сигнатуре `MAKE_OP` | Флаги | Во что превращается в теле `DEFINE_ACLNN_OP` | Примечание |
|---|---|---|---|
| `const aclTensor* имя` | – | `at::Tensor имя;` | загружается через `LOAD_TENSOR` |
| `const aclTensor* имяOptional` | `optional` (по суффиксу) | `at::Tensor имяOptional;` | `LOAD_TENSOR` с `is_optional=1` |
| `const aclTensor* имя` | `out` | `at::Tensor имя;` | загружается, логируется в `end()` |
| `const aclTensor* имя` | `sync` | `at::Tensor имя;` + вызов `exec->имя->sync_after_mutation(имя);` | только для выходов с изменяемой формой |
| `const aclTensor* имя` | `out` (и мы делаем `store`) | `at::Tensor имя;` + **прямой доступ** `exec->имя->store(...)` | поле `exec->имя` остаётся `const aclTensor*`, его не затеняет локальная переменная |
| `const aclScalar* имя` | – | `const at::Scalar& имя = exec->имя->item;` | |
| `optional const aclScalar* имя` | `optional` | `const auto& имя = exec->имя ? std::optional<at::Scalar>(exec->имя->item) : c10::nullopt;` | **не** суффикс в имени, а ключевое слово `optional` |
| `const aclScalar* имя` | `out` | `const at::Tensor& имя = exec->имя->tensor;` | inplace‑скаляр, как в `InplaceAddV3` |
| `const aclIntArray* имя` | – | `const std::vector<int64_t>& имя = exec->имя->data;` | |
| `optional const aclIntArray* имя` | `optional` | `const std::vector<int64_t>& имя = exec->имя ? exec->имя->data : _empty_int_vec;` | |
| `const aclFloatArray* имя` | – | `const std::vector<float>& имя = exec->имя->data;` | |
| `const aclBoolArray* имя` | – | `const std::vector<uint8_t>& имя = exec->имя->data;` | |
| `const aclScalarList* имя` | – | `const std::vector<const aclScalar*>& имя = exec->имя->data;` | |
| `const aclTensorList* имя` | – | `const at::TensorList& имя = exec->имя->aten_tensors();` | ленивая загрузка |
| `aclDataType имя` | – | `const OptionalScalarType имя = toAtenType(exec->имя);` | обёртка `OptionalScalarType` |
| `char* имя` | – | `const std::string& имя = exec->имя;` | ⚠️ временный объект – использовать сразу |
| `float`, `double`, `int64_t`, `bool`, … | – | **подстановка** в теле `MAKE_OP`: `имя` → `exec->имя` | не объявляются отдельно, заменяются текстуально |

#### Флаги и их обозначения в сигнатуре
* `optional` – ключевое слово для скаляров и массивов, **для тензоров не пишется**, вместо этого **имя заканчивается на `Optional`**.
* `out` – пишется перед типом, только для выходов.
* `sync` – пишется перед типом, только для `aclTensor*`, вызывает `sync_after_mutation` после операции.
* Комбинации: `out sync` допустимо? Генератор допускает, обрабатывает оба, но на практике обычно что‑то одно.

### IV. Святая Троица Управления Памятью: `copy_` vs `sync` vs `store`

*   **`copy_`**: Твоя основная рабочая лошадка. Используй **всегда**, когда `torch_npu` уже выделил выходной тензор **правильного размера** и его не надо менять. (`out.copy_(result);`)

*   **`sync`**: Используется, когда форма выхода меняется, **но** `torch_npu` выделил **достаточно большой** буфер (обычно максимально возможный, как для `Nonzero`). Мы просто обрезаем размер через `SYNC_AFTER_MUTATION`. Пример:
    ```cpp
    // Форма меняется, но буфер выделен с запасом.
    at::nonzero_out(out, self); // out.resize_ внутри не требуется
    // SYNC_AFTER_MUTATION обновит desc->dims
    ```

*   **`store`**: Тяжёлая артиллерия. Только для случаев, когда `torch_npu` понятия не имеет о финальном размере и выделяет **нулевой буфер**. Мы **полностью заменяем** буфер и метаданные.
    ```cpp
    // aclnnLogSigmoidForward – уникальный случай, где буфер выхода пуст.
    auto res_buf = at::empty_like(self);
    at::log_sigmoid_forward_out(out, res_buf, self);
    exec->buffer->store(res_buf); // <-- Прямой доступ к aclTensor
    ```

### V. Исключение из Правил: Прямой Доступ к `exec->`
Это твой единственный козырь для обхода автоматической загрузки тензоров. Используй, когда тебе нужен **не ATen-тензор, а сам объект `aclTensor`** (чтобы вызвать его метод `store`).
*   **Правильно:** `exec->buffer->store(...)`
*   **Неправильно:** `exec->from`, `exec->self` (генератор уже превратил их в `at::Tensor`).

### VI. Кодогенератор: Что он делает с твоим `MAKE_OP`
1.  Парсит сигнатуру, определяет типы и модификаторы (`out`, `sync`).
2.  Генерирует `struct Исполнитель` с полями для всех аргументов.
3.  Создаёт `GetWorkspaceSize`, где заполняет эту структуру.
4.  В `DEFINE_ACLNN_OP` вставляет:
    *   `LOAD_TENSOR` для всех `aclTensor*`.
    *   `const at::Scalar&` для `aclScalar*`.
    *   `const auto&` для списков (`aclTensorList*`, `aclIntArray*`).
    *   Замену `имя_аргумента` на `exec->имя_аргумента` для примитивов.
    *   Вызовы `SYNC_AFTER_MUTATION` для всех аргументов с модификатором `sync`.
    *   Обработку исключений и логирование.

### VII. Процесс Создания Новой Операции (Чек-лист)
1.  **Найди сигнатуру** в YAML/JSON/H-файлах.
2.  **Напиши `MAKE_OP`**, следуя синтаксису и таблице типов.
3.  **Определись с памятью**: `copy_` (по умолчанию), `sync` (если форма меняется), `store` (если буфер нулевой).

## 🧠 Логика памяти с учётом генератора

После `LOAD_TENSOR` мы имеем **обычный `at::Tensor`**, можем делать `copy_()`. Если форма не меняется, используем `out.copy_(...)`. Если форма меняется, но буфер выделен с запасом – ставим флаг `sync`, генератор сам добавит `exec->out->sync_after_mutation(out);`. Если буфер был нулевой, мы вручную вызываем `exec->out->store(new_tensor)`, что полностью заменяет память и метаданные.

**Важно:** `store` – не флаг, а действие. Выход с `store` объявляется как `out const aclTensor* out`, затем внутри тела пишем `exec->out->store(res_buf);`. Локальная переменная `out` всё ещё доступна как `at::Tensor`, но мы обращаемся к полю `exec->out` напрямую.

## 🏗️ Что делает генератор с телом `MAKE_OP`

1. Выполняет подстановку **примитивных скаляров** (`alpha` → `exec->alpha`) во всём теле с помощью регулярного выражения.
2. Оборачивает тело в `DEFINE_ACLNN_OP`, который уже содержит:
   - создание `exec`, вызов `exec->start()`,
   - `try/catch` с логированием и `record_op_timing`,
   - автоматический вызов `exec->end()` после тела.
3. Логирование входов/выходов берётся из сгенерированных методов `Executor::start/end`, которые используют перегруженные операторы `<<` из `not_opapi_base.h`.

## 📝 Новый чек-лист для написания операции

1. Найти каноничную сигнатуру в YAML/JSON/H – теперь мы знаем точное имя aclnn‑функции.
2. Записать `MAKE_OP`:
   - Для optional‑тензоров имена заканчиваются на `Optional` (суффикс).
   - Для optional‑скаляров пишем `optional const aclScalar*`.
   - Для sync‑тензоров пишем `sync const aclTensor*`.
   - Примитивы пишем **без** `exec->`.
3. В теле использовать загруженные `at::Tensor`, `at::Scalar`, векторы.
   - Для изменения формы – `sync` в сигнатуре, иначе `copy_`.
   - Для полной замены буфера – оставить `out` и сделать `exec->out->store(...)`.
4. Учесть особенности `char*` (не сохранять ссылку надолго) и `aclDataType` (использовать `OptionalScalarType`).
5. Запустить `not_opapi_gen.py`, собрать, тестировать.
